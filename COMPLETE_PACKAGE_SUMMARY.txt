================================================================================
ARCHITECTURE PATTERN: MVVM
================================================================================

MODEL (Models/*.cs)
  → Plain C# classes representing data entities
  → No UI dependencies
  → Properties: Id, Name, Status, etc.

VIEW (Views/*.xaml)
  → XAML files defining UI structure
  → Data binding to ViewModel
  → No business logic
  → Material Design components

VIEWMODEL (ViewModels/*.cs)
  → Business logic and data preparation
  → INotifyPropertyChanged implementation
  → ObservableCollection for lists
  → Commands for user actions

DATA FLOW:
  User Action → View → ViewModel → Model
  Model Update → ViewModel (notifies) → View (updates UI)

TROUBLESHOOTING QUICK REFERENCE
================================================================================

PROBLEM: Packages won't install
SOLUTION: Check internet connection, update NuGet Package Manager

PROBLEM: Build errors
SOLUTION: Check all files are in correct folders, verify namespaces

PROBLEM: MaterialDesign icons missing
SOLUTION: Verify MaterialDesignThemes package installed, rebuild solution

PROBLEM: Navigation not working
SOLUTION: Ensure Frame element exists, check NavigationButton_Checked method

PROBLEM: Data not showing
SOLUTION: Verify DataContext set in View constructor, check bindings

For detailed troubleshooting, see INSTALLATION_GUIDE.md
