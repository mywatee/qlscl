CONVERSION DETAILS: REACT TO WPF
================================================================================

REACT COMPONENT -> WPF EQUIVALENT
------------------------------------------------------------
React Components        → WPF Pages (in Views/ folder)
TypeScript Models       → C# Classes (in Models/ folder)
React Hooks/State       → ViewModels with INotifyPropertyChanged
React Router            → WPF Frame navigation
Tailwind CSS Classes    → WPF Styles in App.xaml
Material-UI Components  → MaterialDesignThemes controls
Recharts Library        → LiveCharts.Wpf (optional)

SPECIFIC CONVERSIONS
------------------------------------------------------------
<div>                   → <StackPanel>, <Grid>, <Border>
className               → Style attribute
onClick                 → Click event
useState                → Properties with OnPropertyChanged
useEffect               → Constructor or Loaded event
map() function          → ItemsControl.ItemsSource binding
<button>                → <Button>
<input>                 → <TextBox>
<select>                → <ComboBox>
<table>                 → <DataGrid>

DATA BINDING
------------------------------------------------------------
React: {value}          → WPF: {Binding Property}
React: value={state}    → WPF: Text="{Binding Property}"
React: onChange         → WPF: UpdateSourceTrigger=PropertyChanged
React: array.map        → WPF: ItemsSource="{Binding Collection}"

NAVIGATION
------------------------------------------------------------
React Router            → WPF Frame.Navigate()
<Route path="/booking"> → case "Booking": Navigate(new BookingView())
<NavLink to="/staff">   → RadioButton with Checked event

================================================================================
MVVM PATTERN IMPLEMENTATION
================================================================================

MODEL (Data Layer):
- Court, Booking, Customer, Staff, Service classes
- Plain C# classes with properties
- No UI dependencies

VIEW (Presentation Layer):
- XAML files define UI structure
- Data binding to ViewModel properties
- No business logic in code-behind

VIEWMODEL (Logic Layer):
- Inherits from BaseViewModel (INotifyPropertyChanged)
- Contains ObservableCollections for data
- Implements business logic
- Exposes properties for View binding

DATA FLOW:
Model → ViewModel (loads data) → View (displays via binding)
View (user input) → ViewModel (processes) → Model (updates)
